## twitter-to-kafka-service

Микросервис предназначен для получения потока твитов из социальной сети X (ex-Twitter) по заданным ключевым словам.
Есть несколько вариантов, как организовать потоковое получение твитов с ресурса.

**Вариант 1:** Регистрация аккаунта разработчика и получение потока твитов из приложения внутри аккаунта. 
Как оказалось, с 2023 года на бесплатном аккаунте получение стриммингового потока недоступно :angry: 
Тем не менее, конфигурацию я сохранила на всякий случай. Для организации стримминга твитов в папку resources 
следует поместить конфигурационный файл twitter4j.properties. Ключи и токены можно получить в аккаунте 
разработчика. Пример файла:
```properties
debug=true
oauth.consumerKey=*********************
oauth.consumerSecret=******************************************
oauth.accessToken=**************************************************
oauth.accessTokenSecret=******************************************
```

**Вариант 2:** Регистрация аккаунта разработчика и получение данных через REST-запрос с использованием 
Bearer-токена. На бесплатном аккаунте есть ограничения: один запрос в 15 минут, от 10 до 100 результатов 
в запросе, до 1500 твитов в месяц.
